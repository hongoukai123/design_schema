《适配器模式》
1.定义：
    1>适配器模式把一个接口变换成客户端所期待的另一种接口，从而使原本接口不匹配而无法在一起工作的两个类能够在一起工作。
2.用途：
    1>用电器做例子，笔记本电脑的插头一般都是三相的，既除了阴极、阳极外，还有一个地极。而有些地方的电源插座只有两极，
      没有地极，电源的插座与电脑的插头不匹配使得笔记本电脑无法使用。这时候一个三相到两相的转换器(适配器)就能解决此问
      题，而这正像是本模式所做的事情。
3.结构
    1>类的适配器模式
    2>对象的适配器模式
    3>缺省适配器模式
4.用意
    1>适配器模式的用意是改变源的接口，以便于目标接口相容。缺省适配器的用意稍有不同，它是为了方便建立一个不平庸的适配
      器类而提供的一种平庸实现。
    2>在任何时候，如果不准备实现一个接口的所有方法时，就可以使用一个“缺省适配器模式”制造一个抽象类，给出所有方法的平
      庸的具体实现。这样，从这个抽象类再继承下去的子类就不必实现所有的方法了。
